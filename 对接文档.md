## 📋 项目概述

**目标**：实现"手机发指令，PC 自动编程"的远程协作系统

**工作流程**：
```
手机飞书 → 任意自然语言输入（无需/命令格式）
    ↓
PC 端轮询脚本 → SmartRouter 意图识别
    ├─ 管理意图：列出角色/查看状态/激活角色 → 直接回复
    └─ 任务意图：Agent自动匹配 → RAG语义增强 → Claude Code执行
    ↓
智能分段结果 → 回复到飞书群聊
```

## 📝 已修复的 Bug（供参考）

### Bug 1：缺少 `container_id_type` 参数

**现象**：调用 `GET /im/v1/messages` 时报错 `99992402 field validation failed`

**原因**：飞书 API 要求必须传 `container_id_type` 参数

**修复**：在 `run_feishu_poll.py` 第 114 行添加：
```python
params={"container_id": CHAT_ID, "container_id_type": "chat", "page_size": 5}
```

---

### Bug 2：机器人循环回复自己发的消息

**现象**：机器人回复消息后，脚本又检测到这条消息，再次回复，形成死循环

**原因**：没有过滤掉机器人自己发的消息

**修复**：在 `run_feishu_poll.py` 第 138-141 行添加：
```python
# 跳过机器人自己发的消息
sender_id = msg.get("sender", {}).get("id", "")
if sender_id == APP_ID:
    continue
```

---

### Bug 3：`@机器人 /run` 格式无法识别

**现象**：用户发送 `@_user_1 /run 你好`，脚本无法识别

**原因**：指令开头是 `@_user_1`，不是 `/run`

**修复**：在 `run_feishu_poll.py` 第 149-152 行改用正则匹配：
```python
import re
match = re.search(r'/run\s+(.*)', content, re.DOTALL)
if not match:
    continue
instruction = match.group(1).strip()
```
